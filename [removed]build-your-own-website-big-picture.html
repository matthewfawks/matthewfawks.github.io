How to Build Your Own Website
Big Picture Guide

Here I have put together a brief, bullet-pointed guide as an overarching view on how to create your own website. In basic, you will need your own domain name (ex. "matthewfawks.com" from something like Namecheap), somewhere to host the site (ex. GitHub Pages), and something to edit the site with (ex. Atom). To start to get a grasp on how to even begin, the best thing to do would be to familiarize yourself with the syntax of the languages involved and to see some guides/templates to follow along and get acquainted with. Next, you'll want the tools necessary to editing/organizing the source-code for your site. Finally, you'll want to get a domain name that is yours, choose a spot to host your source-code, and configure the domain name within your chosen host. After these steps are followed, you will be well on your way to diving into the very extensive world of front-end web development!

Step 1: Familiarize Yourself

The primary languages you will need to know are HTML (Hypertext Markup Language), CSS (Cascading Style Sheets), and JS (JavaScript). HTML provides the content and structure to web applications, CSS provides the styling and "dressing up" of HTML, and JS provides the logic and interactivity in web applications. Below, I have provided a few helpful resoures to both get acquainted with these languages and start your journey towards hosting your first web page.

Step 2: Get the Necessary Tools

In order create and edit the source-code of your webpage, you will need some text editing software and some way to push those changes to where you are hosting. For your text editor, you could use an online IDE such as repl.it (this particular one can be used for well beyond just HTML/CSS/JS) or you can download some software and locally edit your files on your machine. I personally prefer the latter and use Atom to edit my source-code and GitHub Desktop to manage my changes and push them to the host. There a lot of ways to do this out there, you just have to do some research and figure out which option best meets your preferences.

Software to Download (for text/source-editing):

    • GitHub Desktop
    • Text/source-code editing software (a fantastic option would be Adobe Dreamweaver (~$21/mo.), but a free alternative is something like Atom or Sublime Text, and there are plenty of other great ones out there too - some other examples on Mac OS could be found here)

Step 3: Get Your Domain and Host Your Site

After determining how you are going to edit your source-code, you now need to purchase your domain name and determine where you want to host your site. For hosting, you generally have three options. First, you can do traditional web hosting through something like Namecheap or GoDaddy (also where you can get your domain name from), but these only alot a set amount of server capacity and probably charge more than you would like to pay for just your first site. Second, you could use cloud services like Amazon Web Services, but this is way too much firepower for a simple front-end/static webpage (ie. a page that's just using HTML/CSS/JS). Third, and the way I do it, is through GitHub Pages, which is a free and excellent way to learn how to host your site. Whichever you choose will depend on your desired use and budget, so just go with the option that best fits your needs.

Logins/Accounts to Have (for hosting):

    • Namecheap (for domain name purchasing/handling)
      • Alternative would be GoDaddy
    • GitHub Account (for the website hosting)
      • An alternative basic hosting site if you don’t want to purchase a domain right now could be altervista, but it keeps “.altervista.com” in the URL
      • Namecheap and GoDaddy also have the ability to host for a cost, but this wouldn’t be necessary for a basic site
      • A pretty nice guide comparing other hosting services outside of these few could be found here.

Note: Base-Level Costs

    • Domain Name Renewal: ~$11/yr. (w/ Namecheap and greatly depends on what domain you select)
    • Pages Hosting: Free (w/ Github Pages)
    • Total = ~$11/year

*An alternative to this manual process would be to use a site like Carrd (~$9/yr. for the most basic plan or ~$19/yr. if you want to use a custom domain) or Wix (~$13/mo. for most basic plan + the ~$10/yr. price for domain name ownership, or $0/mo. (free) if want to deal with Wix branding on the site, “.wix.com” in the URL, and limited features) for hassle-free website creation and hosting. But where's the learning in that?
